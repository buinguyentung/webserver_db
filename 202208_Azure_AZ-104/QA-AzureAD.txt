You have an Azure Active Directory (Azure AD) tenant that has the contoso.onmicrosoft.com domain name.
You have a domain name of contoso.com registered at a third-party registrar.
You need to ensure that you can create Azure AD users that have names containing a suffix of @contoso.com.
Which three actions should you perform in sequence?
1. Add a record to the public contoso.com DNS zone
2. Add an Azure AD tenant
3. Configure company branding
4. Create an Azure DNS zone
5. Add a custom name
6. Verify the domain
	A. 5 - 1 - 6
	B. 4 - 5 - 1
	C. 1 - 5 - 6
	D. 2 - 4 - 6
Explanation
Correct Answer:  5 -> 1 -> 6
1. Add the custom domain name to your directory
2. Add a DNS entry for the domain name at the domain name registrar
3. Verify the custom domain name in Azure AD

//=== TEST-4
You have the Azure management groups shown in the following table:
	Tenant Root group
	MG11		Tenant Root group
	MG12		Tenant Root group
	MG21		MG11
You add Azure subscriptions to the management groups as shown in the following table:
	Subscription1	MG21
	Subscription2	MG12
You create the Azure policies shown in the following table:
	Not allowed resource types		VirtualNetworks		Tenant Root group
	Allowed resource types			VirtualNetworks		MG12

* You can create a virtual network in Subscription1
Explanation
Correct Answer: B (NO)
Subscription1 is a member of ManagementGroup21, ManagementGroup21 is a member of ManagementGroup11, ManagementGroup11 is a member of the Tenant Root Group, The Tenant Root group has Not allowed resource types for virtual network.

* You can create a virtual machine in Subscription2
Explanation
Correct Answer: B (NO)
Allowed Resource Type (Deny): Defines the resource types that you can deploy. Its effect is to deny all resources that aren't part of this defined list.
ManagementGroup12 is in Subscription2 and in the Scope of the 2nd policy. Allowed resource types are virtualNetworks. This means all other resources are denied by default.

* You can add Subscription1 to ManagementGroup11
Explanation
Correct Answer: B (NO)
You cannot ADD Subscription1 to ManagementGroup11, but you can MOVE Subscription1 from ManagementGroup21 to ManagmentGroup11. Subscriptions can only be a member of ONE ManagementGroup at a time.
